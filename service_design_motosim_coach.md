# MotoSim Coach 서비스 설계안

## 1. 서비스 개요

MotoSim Coach는 오토바이 면허 및 안전교육 시뮬레이션 프로그램에 내장되는 AI 주행 코칭 모듈이다.

학습자가 시뮬레이터에서 주행하면, 시스템은 주행 화면을 semantic segmentation 모델로 분석하여 도로, 차선, 움직이는 객체, 주변 라이더, 내 오토바이 영역을 구분한다. 이를 바탕으로 차선 유지, 전방 위험 객체 인지, 주행 가능 영역 이탈 가능성을 평가하고 학습자에게 실시간 피드백과 주행 후 리포트를 제공한다.

## 2. 문제 정의

오토바이는 자동차보다 차체가 작고 외부 환경에 직접 노출되기 때문에, 도로 상태와 주변 객체 변화에 민감하다. 특히 야간 주행에서는 시야 확보가 어렵고, 차선과 도로 경계가 불명확해질 수 있다.

초보 라이더는 실제 도로에 나가기 전에 다음과 같은 능력을 충분히 훈련할 필요가 있다.

- 차선 중앙을 안정적으로 유지하는 능력
- 전방 차량, 보행자, 라이더 등 위험 객체를 인지하는 능력
- 주행 가능 영역과 주행 불가 영역을 구분하는 능력
- 야간 또는 저시야 환경에서 감속과 회피 판단을 하는 능력

MotoSim Coach는 실제 도로 주행 전에 이러한 상황 판단 능력을 시뮬레이션 환경에서 반복적으로 훈련하도록 돕는 것을 목표로 한다.

## 3. 타깃 사용자

- 오토바이 면허 취득을 준비하는 학습자
- 야간 주행 경험이 부족한 초보 라이더
- 배달, 통근 등 반복 주행을 준비하는 라이더
- 오토바이 안전교육 기관 또는 운전면허 교육기관
- 시뮬레이터 기반 안전교육 콘텐츠를 운영하는 기관

## 4. 서비스 입력과 출력

### 입력

- 오토바이 시뮬레이터 주행 화면
- 시뮬레이터에서 추출한 이미지 프레임
- 주행 종료 후 저장된 훈련 영상

### 출력

- semantic segmentation 결과
- 위험 상황 피드백
- 차선 유지 평가
- 전방 위험 객체 감지 결과
- 주행 후 코칭 리포트
- 위험 구간 타임라인

## 5. 모델 출력 Class

모델은 입력 이미지의 각 픽셀을 아래 class 중 하나로 분류한다.

| Class ID | Class Name | 의미 |
|---:|---|---|
| 0 | Undrivable | 주행 불가 영역 |
| 1 | Road | 주행 가능한 도로 영역 |
| 2 | Lane Mark | 차선 |
| 3 | Moveable | 차량, 보행자 등 움직이는 객체 |
| 4 | Rider | 주변 라이더 |
| 5 | My bike | 내 오토바이 영역 |

## 6. 핵심 기능

### 6.1 차선 유지 코칭

Lane Mark와 My bike 영역을 활용하여 학습자가 차선 중앙을 안정적으로 유지하고 있는지 분석한다.

예시 피드백:

- 차선 중앙에서 벗어나는 경향이 있습니다.
- 오른쪽 차선에 가까워지고 있습니다.
- 차선 인식이 어려운 구간입니다. 감속이 필요합니다.

### 6.2 전방 위험 객체 인지 코칭

Moveable 또는 Rider 영역이 Road 영역 내부나 주변에 나타날 경우 위험 후보로 판단한다.

예시 피드백:

- 전방 이동 객체와의 거리가 가깝습니다.
- 주변 라이더 또는 차량 접근 가능성이 있습니다.
- 위험 객체가 주행 가능 영역에 진입했습니다.

### 6.3 주행 가능 영역 분석

Road 영역의 위치와 면적 변화를 분석하여 학습자가 안정적인 주행 가능 영역 안에 있는지 판단한다.

예시 피드백:

- 주행 가능 영역이 좁아지는 구간입니다.
- 도로 경계에 가까워지고 있습니다.
- 도로 외 영역으로 치우치는 경향이 있습니다.

### 6.4 야간 시야 위험 구간 탐지

Undrivable 영역이 과도하게 넓거나 Road / Lane Mark 검출이 불안정한 구간을 저시야 위험 구간으로 표시한다.

예시 피드백:

- 야간 시야가 불안정한 구간입니다.
- 차선과 도로 경계가 명확하지 않습니다.
- 감속 후 주변 객체를 확인하는 것이 필요합니다.

### 6.5 주행 후 리포트 제공

훈련 종료 후 학습자의 주행 패턴을 요약한 리포트를 제공한다.

리포트 항목 예시:

- 안전 주행 점수
- 차선 유지 점수
- 위험 객체 근접 횟수
- 주행 가능 영역 이탈 가능 구간
- 야간 시야 위험 구간
- 개선이 필요한 장면 캡처
- segmentation overlay 이미지

## 7. Class별 서비스 활용 방식

| Class | 서비스 활용 |
|---|---|
| Undrivable | 도로 외부 영역과 주행 불가 영역 판단 |
| Road | 주행 가능 영역 분석 |
| Lane Mark | 차선 유지 여부 판단 |
| Moveable | 차량, 보행자 등 동적 위험 객체 감지 |
| Rider | 주변 이륜차 또는 라이더 감지 |
| My bike | 학습자의 오토바이 위치 기준점 |

## 8. 서비스 사용 흐름

1. 학습자가 오토바이 시뮬레이터 주행을 시작한다.
2. 시뮬레이터 화면 프레임이 AI 코칭 모듈에 입력된다.
3. semantic segmentation 모델이 프레임의 class mask를 예측한다.
4. 시스템이 Road, Lane Mark, Moveable, Rider, My bike 영역을 분석한다.
5. 위험 상황이 감지되면 실시간 코칭 메시지를 제공한다.
6. 주행 종료 후 위험 구간과 개선점을 포함한 리포트를 생성한다.

## 9. 기대 효과

- 실제 도로 주행 전 위험 상황 판단 능력 훈련
- 초보 라이더의 차선 유지 및 감속 판단 능력 개선
- 시뮬레이터 기반 오토바이 안전교육 콘텐츠 고도화
- 강사의 주관적 평가를 보조하는 정량적 피드백 제공
- 반복 훈련을 통한 학습자의 위험 인지 능력 향상

## 10. 기술적 구성

### 데이터

- 기본 데이터: 오토바이 야간 주행 semantic segmentation 데이터
- 외부 데이터: CamVid 도로 주행 segmentation 데이터
- 외부 데이터 활용 방식: 프로젝트 class 체계로 remapping 후 pretraining에 사용

### 모델

- semantic segmentation 모델 사용
- 후보 모델:
  - U-Net
  - ResNet 기반 segmentation model
  - SegFormer-B0
  - SegFormer-B2

### 평가 지표

- mIoU
- class별 IoU
- Dice score
- Pixel accuracy

## 11. 현재 한계점

- 기본 오토바이 데이터셋이 200장으로 적어 일반화 성능에 한계가 있다.
- 외부 데이터는 자동차 주행 시점이 많아 오토바이 시뮬레이터 환경과 차이가 있다.
- CamVid에는 My bike class가 없어 해당 class를 외부 데이터로 보강하기 어렵다.
- 실제 시뮬레이터 내장형 서비스로 확장하려면 추론 속도와 실시간 피드백 지연 시간을 추가로 검증해야 한다.
- 현재 단계에서는 위험 판단 로직이 rule-based 형태이므로, 향후 더 정교한 시간적 분석이 필요하다.

## 12. 향후 개선 방향

- 야간, 우천, 안개 등 다양한 환경 데이터 추가
- 시뮬레이터에서 직접 생성한 synthetic data 활용
- frame 단위 분석을 영상 단위 위험 점수로 확장
- 위험 객체와 My bike 사이의 상대적 위치 기반 코칭 고도화
- 강사 모드 dashboard 추가
- 실시간 추론을 위한 모델 경량화 및 최적화

## 13. 발표용 요약 문장

MotoSim Coach는 오토바이 면허 및 안전교육 시뮬레이터에 내장되는 AI 주행 코칭 모듈이다. Semantic segmentation을 통해 도로, 차선, 움직이는 객체, 주변 라이더, 내 오토바이 영역을 분석하고, 학습자에게 실시간 피드백과 주행 후 리포트를 제공함으로써 실제 도로 주행 전 위험 상황 인지 능력을 훈련하도록 돕는다.
